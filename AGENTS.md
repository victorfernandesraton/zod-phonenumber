# Project zod-phonenumber

## General definition and goal

Library to provide phone number validation for [zod](https://zod.dev) schemas using [libphonenumber-js](https://www.npmjs.com/package/libphonenumber-js). Extends the zod ecosystem with a `z.phone()` method that validates and transforms phone number strings into structured `PhoneNumber` objects.

### Implementation Details

The library uses zod v4's structural type system. The `z.phone()` function extends the zod namespace by spreading all zod exports and adding the `phone` factory function.

**`phone.ts`** implements `createPhoneSchema()` which:
1. Creates `z.string()` as the base input type
2. Adds `.refine()` for validation: parses with `libphonenumber-js`, checks `isPossible()`, validates DDI/DDD/country constraints
3. Adds `.transform()` to convert validated string into `PhoneNumber` object

The returned schema includes chainable methods via `Object.assign`:
- `.country(code)` — filter by country code (e.g. `'BR'`, `'US'`)
- `.ddi(code)` — filter by international calling code (e.g. `'+55'`, `'55'`)
- `.ddd(code)` — filter by Brazilian area code (implicitly sets country to `'BR'` if not already set)

## API Reference

### `z.phone(country?)`

Creates a phone number schema. Accepts optional default country code.

```typescript
import { z } from 'zod-phonenumber'

// Basic: validates any international phone number → returns PhoneNumber
const phone = z.phone().parse('+12133734253')
phone.country         // 'US'
phone.number          // '+12133734253'
phone.nationalNumber  // '2133734253'
phone.isValid()       // true (requires max metadata)
phone.isPossible()    // true

// Constructor argument: sets default country
const brPhone = z.phone('BR')
brPhone.parse('+5511999988888')  // ok
brPhone.parse('+12133734253')    // throws (wrong country)
```

### `.country(code: string)`

Restricts validation to a specific country.

```typescript
z.phone().country('BR').parse('+5511999988888')   // ok
z.phone().country('BR').parse('+12133734253')     // throws
```

### `.ddi(code: string)`

Restricts validation to a specific international calling code (DDI). Accepts with or without `+` prefix.

```typescript
z.phone().ddi('+55').parse('+5511999988888')   // ok
z.phone().ddi('+55').parse('+12133734253')     // throws
z.phone().ddi('55').parse('+5511999988888')    // ok (without + prefix)
```

### `.ddd(code: string)`

Restricts validation to a specific Brazilian area code (DDD). Automatically sets country to `'BR'` if not already constrained. Not applied to non-Brazilian numbers.

```typescript
z.phone().ddd('11').parse('+5511999988888')    // ok (SP area code)
z.phone().ddd('11').parse('+5521999988888')    // throws (RJ area code)
```

### Chaining

```typescript
// All methods are chainable
z.phone().country('BR').ddd('11')
z.phone().ddi('+55').ddd('11')
```

### Integration with z.object()

```typescript
const schema = z.object({
  name: z.string(),
  phone: z.phone().country('BR').ddd('11'),
})

const result = schema.parse({ name: 'Joao', phone: '+5511999988888' })
result.phone instanceof PhoneNumber  // true
result.phone.country                 // 'BR'
```

## Output Type

The schema transforms the input string into a `PhoneNumber` object from `libphonenumber-js`. The `PhoneNumber` class provides:

| Property/Method | Description |
|-----------------|-------------|
| `.country` | ISO country code (e.g. `'BR'`, `'US'`) |
| `.number` | E.164 formatted number (e.g. `'+5511999988888'`) |
| `.nationalNumber` | National significant number (e.g. `'1199998888'`) |
| `.countryCallingCode` | Country calling code (e.g. `'55'`) |
| `.isPossible()` | Validates phone number length |
| `.isValid()` | Validates both length and digit patterns (max metadata) |
| `.getType()` | Phone number type (max metadata) |
| `.formatInternational()` | International format (e.g. `'+55 11 99999 8888'`) |
| `.formatNational()` | National format (e.g. `'(11) 99999-8888'`) |
| `.getURI()` | Tel URI (e.g. `'tel:+5511999988888'`) |

## DDD (Brazilian Area Code) Behavior

The DDD (Discagem Direta a Distancia) is a 2-digit area code system specific to Brazil. Each Brazilian state has one or more assigned DDD codes (e.g. `11` for Sao Paulo, `21` for Rio de Janeiro).

- `.ddd(code)` only filters when the parsed number has `country === 'BR'`
- When `.ddd()` is called without a prior `.country()` or `.ddi()` constraint, country is implicitly set to `'BR'`
- For non-Brazilian numbers, the DDD filter is a no-op

## Notes for AI Agents

1. **This is a zod v4 extension** — v4 uses structural types, not class-based ZodType hierarchy like v3
2. **The re-export pattern** (`import { z } from 'zod-phonenumber'`) is used instead of monkey-patching because ESM module imports are immutable
3. **`libphonenumber-js` default import** (`import parsePhoneNumber from 'libphonenumber-js'`) returns `undefined` on invalid input. The named `parsePhoneNumber` export is deprecated and throws `ParseError`
4. **Tests run via `tsx --test`** — TypeScript is executed directly, no pre-compilation needed for testing
5. **Strict TypeScript**: All `as` assertions must be explicit intersection types, never `any` or `@ts-ignore`
6. **`defaultCountry` in parsePhoneNumber()** is ignored for numbers starting with `+` — explicit country check must be done after parsing
7. **The `min` metadata set** (default) only supports `isPossible()`. For `isValid()` and `getType()`, import from `libphonenumber-js/max`
